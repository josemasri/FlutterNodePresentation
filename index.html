Funcionamiento de Servlet

  1. El cliente envía una petición al servidor web.
  2. El servidor recibe la petición.
  3. El servidor delega la petición a su correspondiente Servlet.
  4. El Servlet realiza un procesamiento y genera una respuesta.
  5. El servlet envía esta respuesta al servidor web.
  6. El servidor web envía la respuesta de nuevo al cliente y navegador del cliente le despliega la respuesta en su pantalla al usuario.

Estructura de carpetas

  1. ".dart_tool/": Librerías de pub instaladas
  2. ".idea/": Archivos y configuraciones para el IDE de Android Studio.
  3. "android/": Proyecto completo de Android nativo.
  4. "ios/": Proyecto completo de IOS nativo.
  5. "lib/": Nuestro código fuente dart
  6. "test/": Realizar pruebas automátizadas
  7. "pubspec.yaml": Paquetes de terceros que se desean utilizar, configuraciónes, fuentes, imágenes estáticas.

NestJS

¿Porque Nest.js?

¿Porque usar Nest.js?

  • Posee una documentación muy buena, con ejemplos que realmente te ayudan a entender los conceptos.
  • Arquitectura: Nest tiene una arquitectura de proyecto que proporciona una capacidad de prueba sin esfuerzo , escalabilidad y capacidad de mantenimiento.

¿Porque usar Nest.js?

  • Ya que utiliza TypeScript y los conceptos básicos de Angular, los desarrolladores de Angular pueden aprenderlo rápidamente y podrán crear backends para sus aplicaciones Angular sin tener que recurrir a otros frameworks del lado del servidor

Introducción al uso de NestJs

Instalación

Instalar la CLI de Nestjs


						$> npm i -g @nestjs/cli

Crear un proyecto


						$> nest new project-name

Estructura de carpetas

  • "node_modules/:" Dependencias de node
  • "src/:" Nuestra aplicación
  • "package.json:" Configuraciones de npm y dependencias
  • "tsconfig.json:" Configuraciones de typescript

Funcionamiento general


                import { NestFactory } from '@nestjs/core';
                import { AppModule } from './app.module';
  
                async function bootstrap() {
                  const app = await NestFactory.create(AppModule);
                  await app.listen(3000);
                }
                bootstrap();

Módulo


                @Module({
                  imports: [
                  TasksModule, 
                  MongooseModule.forRoot(
                    'mongodb://localhost/tasks', 
                    {
                      useNewUrlParser: true, 
                      useUnifiedTopology: true
                    }
                  )],
                  controllers: [AppController],
                  providers: [AppService],
                })
                export class AppModule {}

Clase anotada con decorador @Module(), sirve para organizar la estructura de la aplicación.

Controlador


                @Controller('cats')
                export class CatsController {
                  @Get()
                  findAll(): string {
                    return 'This action returns all cats';
                  }
                }

Clase anotada con decorador @Controller(), encargados de manejar las peticiones de los clientes.

Proveedores


                @Injectable()
                export class CatsService {
                  private readonly cats: Cat[] = [];
                  create(cat: Cat) {
                    this.cats.push(cat);
                  }
                  findAll(): Cat[] {
                    return this.cats;
                  }
                }

Clase anotada con decorador @Injectable(), que permite ser invocada desde cualquier modulo donde se importe y conserva la misma instancia (Singleton). Se utiliza para trabajar con la lógica de negocio.

Laravel vs Hibernate

Pros

  • Curva de aprendizaje relativamente Baja (en comparación con otros framework Php).
  • Es modular y con una amplio sistemas de paquetes y drivers lo que lo hace fácil, robusta y segura.

Contras

  • La principal desventaja que notamos cuando comenzamos a utilizarlo fue su fuerte orientación hacia la programación estática (usa una abusiva cantidad de clases y métodos estaticos).

Hibernate vs Laravel

Pros

  • Documentación de la comunidad de Hibernate, la más amplia alrededor del Object Mapping
  • Permite trabajar con la base de datos por medio de entidades en vez de Querys.

Contras

  • No ofrece toda la funcionalidad que ofrecería tirar consultas nativas.
  • Puede representar una curva de aprendizaje mas grande.
